How LocalShop Works

The "Big Idea"

This project, "LocalShop," is a full-stack, dual-sided web application. It solves the problem for both customers (who can't find items) and shop owners (who can't easily show their inventory).

1. The Private Shop Owner App (The "Admin" Side)

This is a secure area where shop owners log in to manage their business.

  • Secure Login: Owners register and log in. An owner can only see their own shops and data.
  • Multi-Shop Management: An owner can create, edit, and delete multiple shop profiles.
  • Item Management: For each shop, the owner can add items, including photos, descriptions, and specific quantities (e.g., "10 kg", "5 units").
  • Location: Owners can set their shop's exact location on a map using their current GPS or by dragging a pin.

2. The Public Customer App (The "Search" Side)

This is the public-facing site where any customer can find items without needing to log in.

  • Public Search: A customer can search for any item (e.g., "Apples").
  • Database Search: The app searches the database for all items matching that name that are in stock (units > 0).
  • List & Map View: The results are shown in two ways:
    1. A scrollable list of items, showing the item name, stock, and which shop has it.
    2. All shops from the search results are plotted as pins on a Leaflet map.
  • Product & Location Pages: Customers can click for details and then click "Locate Shop" to see a map with a route from their location to the shop.

3. The Backend & Security Model (The "Brain")

The entire app runs on a Supabase backend with a PostgreSQL database. The most important feature is our "Asymmetric" security:

  • Public Read: ANY guest or customer is allowed to READ (SELECT) all shop and item data. This is what makes the search work.
  • Private Write: A user can ONLY WRITE (INSERT, UPDATE, DELETE) data if they are the logged-in owner of that specific shop. This keeps the owners' data safe from each other.
